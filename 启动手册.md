# 攻防演练系统 - 启动手册

## 项目概述

本系统是一个攻防演练分数管理系统，包含以下功能：
- 管理员后台：队伍管理、用户管理、分数管理、文档管理、文件上传日志
- 普通用户端：查看排名、上传文档
- 大屏展示：实时展示队伍分数及排名

## 技术栈

- **前端**: React + TypeScript + Vite + TailwindCSS 3
- **后端**: Node.js + Express + TypeScript
- **数据库**: SQLite（持久化存储）
- **文件上传**: multer（文件存储在 `uploads/` 目录）

## 环境要求

- Node.js >= 18.x
- npm >= 9.x

## 快速启动

### 1. 安装依赖

```bash
cd 打分
npm install
```

### 2. 启动服务

#### 方式一：同时启动前后端（推荐）

```bash
npm run dev
```

#### 方式二：分别启动

**启动后端服务**（端口 3001）：
```bash
npm run server:dev
```

**启动前端服务**（端口 5173）：
```bash
npm run client:dev
```

## 数据库管理

### 数据文件位置

数据库文件位于项目根目录：`data.db`

### 数据存储内容

数据库包含以下表：

| 表名 | 描述 |
|------|------|
| teams | 队伍信息 |
| users | 用户信息（包含密码） |
| scores | 分数记录（支持正负分） |
| documents | 文档记录 |
| upload_logs | 文件上传日志 |

### 手动修改数据

可以使用 SQLite 工具直接操作数据库：

```bash
# 安装 sqlite3 命令行工具（可选）
npm install -g sqlite3

# 打开数据库
sqlite3 data.db

# 查看所有队伍
sqlite> SELECT * FROM teams;

# 查看所有用户
sqlite> SELECT * FROM users;

# 查看分数记录
sqlite> SELECT * FROM scores;

# 添加分数（支持负分）
sqlite> INSERT INTO scores (id, team_id, points, description, timestamp) VALUES ('100', '1', -50, '违规扣分', datetime('now'));

# 修改用户密码
sqlite> UPDATE users SET password = 'newpassword' WHERE username = 'admin';

# 退出
sqlite> .quit
```

### 常见数据库操作

#### 1. 备份数据库
```bash
cp data.db data.db.backup
```

#### 2. 恢复数据库
```bash
cp data.db.backup data.db
```

#### 3. 重置数据库（删除所有数据）
```bash
rm data.db
npm run server:dev  # 重启服务会自动创建新数据库
```

#### 4. 查看数据库大小
```bash
ls -lh data.db
```

## 文件存储位置

用户上传的文件保存在项目根目录的 `uploads/` 文件夹中：

```
打分/
├── uploads/           # 上传文件目录（文件名保持原始名称）
├── data.db           # SQLite数据库文件
├── api/              # 后端代码
├── src/              # 前端代码
└── ...
```

### 文件命名规则

- **文件名保持原始名称**：用户上传的文件名会原封不动地保存
- **服务器存储路径**：`uploads/原始文件名`
- **下载时**：管理员下载的文件名与上传时完全一致

## 常见问题

### Q1: 服务启动失败

**原因**: 端口被占用或依赖未安装

**解决**:
```bash
# 确保依赖已安装
npm install

# 检查端口占用
netstat -ano | findstr :3001  # Windows
lsof -i :3001                 # Linux/Mac

# 杀死占用进程
taskkill /F /PID <进程ID>      # Windows
kill -9 <进程ID>               # Linux/Mac
```

### Q2: 文件上传失败

**原因**: uploads目录不存在或权限不足

**解决**:
```bash
# 创建目录
mkdir uploads

# 设置权限（Linux/Mac）
chmod 755 uploads
```

### Q3: 数据库损坏

**原因**: 非正常关闭或磁盘空间不足

**解决**:
```bash
# 备份损坏的数据库
cp data.db data.db.broken

# 删除损坏的数据库
rm data.db

# 重启服务自动创建新数据库
npm run server:dev
```

### Q4: 中文文件名显示乱码

**已解决**: 系统现在完整支持中文文件名，下载时保持原始文件名。

## 功能说明

### 1. 分数管理（支持负分）

现在可以给队伍添加负分，例如：
- 成功攻击：+100分
- 违规操作：-50分
- 防守成功：+80分
- 未能防御：-30分

### 2. 文件上传日志

系统记录所有文件操作：
- 上传时间
- 上传者
- 所属队伍
- 文件名
- 文件大小
- 操作类型（上传/删除）

### 3. 中文文件名支持

- 上传：保持原始中文文件名
- 存储：文件原名存储在uploads目录
- 下载：下载的文件名与上传时完全一致
- 显示：前端正确显示中文文件名

## 注意事项

1. **数据持久化**: 使用SQLite数据库，重启服务后数据不会丢失。

2. **文件存储**: 上传的文件保存在 `uploads/` 目录，重启服务不会丢失。

3. **数据库位置**: 数据库文件为 `data.db`，请定期备份重要数据。

4. **端口配置**: 默认端口为 3001（后端）和 5173（前端），可在 `api/server.ts` 和 `vite.config.ts` 中修改。

5. **安全考虑**: 当前密码为明文存储，生产环境应使用 bcrypt 加密。
